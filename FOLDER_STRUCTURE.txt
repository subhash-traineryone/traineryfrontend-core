COMPONENT CATEGORIES EXPLANATION:
=================================

CHART COMPONENTS:
- chart/ = Organization chart components (hierarchical employee structure)
- charts/ = Data visualization charts (age, gender, type distributions)

COMMON COMPONENTS:
- common/ = Reusable UI components used across the application
- ui/ = Shadcn UI component library components

EMPLOYEE COMPONENTS:
- employee/ = Employee listing and management components
- modals/ = Employee-related modals and sidebars

LAYOUT COMPONENTS:
- layout/ = Main application layout components (sidebar, navigation)
- maps/ = Map and location-based components

STATISTICS COMPONENTS:
- stats/ = Statistics cards and metrics display components

WORKFORCE COMPONENTS:
- workforce/ = Workforce management specific components

PAGE COMPONENTS:
- pages/ = Main page components and complex form components
- employee-details/ = Employee detail page sections
- forms/ = Form components and tabs

IMPORT PATTERNS:
===============

From components folder:
import { ComponentName } from '../components';

From specific component category:
import { ComponentName } from '../components/charts';
import { ComponentName } from '../components/employee';
import { ComponentName } from '../components/modals';

From pages folder:
import { ComponentName } from '../pages/employee-details';
import { ComponentName } from '../pages/forms';

BARREL EXPORTS:
==============

Each folder has an index.ts file that exports all components in that folder.
Main components/index.ts exports all categories for easy importing.

FOLDER NAMING CONVENTIONS:
=========================

- chart/ = Singular, for organization charts
- charts/ = Plural, for data visualization charts
- common/ = Reusable components
- employee/ = Employee management
- layout/ = Layout components
- maps/ = Map components
- modals/ = Modal and sidebar components
- stats/ = Statistics components
- ui/ = UI library components
- workforce/ = Workforce management

This structure ensures clear separation of concerns and easy navigation.
